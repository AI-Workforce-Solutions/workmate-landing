Chương 1Giới thiệu

Workmate AI là ứng dụng desktop (Electron) dùng để điều khiển và tương tác với các AI agent trên nền tảng OpenClaw. Hiểu đơn giản: Workmate AI là giao diện đồ họa cho OpenClaw — quản lý agent, chat, tự động hóa, kết nối kênh nhắn tin, tất cả từ một cửa sổ duy nhất.

📌 Về nguồn gốc: Fork từ Neurons-ai/CrawBot, bổ sung thêm các tính năng cá nhân: RAG, LocalLLM, QuickSearch, WorkflowBuilder. Những tính năng này không có trong bản upstream gốc.

Chương 2Kiến trúc hệ thống

Workmate AI hoạt động theo mô hình 3 tầng: Renderer (UI) → Main Process (điều phối) → OpenClaw Gateway (AI engine).

┌─────────────────────────────────────────────────────────┐ │ Workmate AI Desktop App │ │ │ │ Renderer (React 19 + Zustand) Main Process (Node) │ │ src/pages/ src/stores/ ←IPC→ electron/main/ │ │ ipc-handlers.ts │ │ │ │ │ │ └──────────┬───────────┘ │ │ │ WebSocket JSON-RPC │ │ ▼ │ │ OpenClaw Gateway (:18789) │ │ AI Engine · Agent Runner · Skills │ └─────────────────────────────────────────────────────────┘

Các thành phần chính

Thành phầnVị tríVai trò
Renderersrc/Giao diện React, state Zustand
Main Processelectron/main/Window, tray, IPC handlers, updater
Gateway Managerelectron/gateway/Khởi động OpenClaw, WebSocket, OAuth patcher
Utilselectron/utils/Config, paths, provider registry, UV/Python
OpenClawChild processAI engine thực sự — chạy agent, tools, skills
💡 Ports: Dev :5173 (Vite) · Gateway :18789 · Production :23333

Chương 5Chat với Agent

Tính năng cốt lõi — tương tác với AI agent theo thời gian thực, xem quá trình suy nghĩ và thực thi tool.

Extended Thinking

Bật toggle Extended Thinking trên toolbar → agent hiển thị block "suy nghĩ" trước khi trả lời. Click vào block để mở rộng / thu gọn.

📌 Chỉ khả dụng với Claude 3.5 Sonnet trở lên.

Tool Execution

Agent gọi tool → Workmate AI hiển thị tool call card: tên tool · input · output · trạng thái (running / done / error).

Tools Auto-Approve: Settings → Chat → bật để agent tự chạy tool không cần xác nhận. Tắt để kiểm soát từng bước.

Token Counter

Toolbar hiển thị số token thực tế (input / output / cache) theo thời gian thực — theo dõi chi phí API.

Dừng Agent

  • Abort Stream — dừng response đang stream, agent vẫn chạy ngầm
  • Stop Agent — dừng hoàn toàn agent run hiện tại

Cron Expression tham khảo

* * * * *      # Mỗi phút
0 * * * *      # Mỗi giờ

Chương 11RAG — Tri thức từ tài liệu Personal Feature

RAG (Retrieval-Augmented Generation) cho phép agent tham chiếu nội dung từ tài liệu bạn cung cấp — PDF, TXT, Markdown — thay vì chỉ dùng kiến thức training.

Tài liệu (PDF / TXT / MD) │ ▼ Embedding (vector hóa nội dung) Vector Database (local) │ ▼ Agent query: "tìm đoạn liên quan" Relevant chunks → thêm vào context của AI │ ▼ AI trả lời dựa trên nội dung thực tế của tài liệu

Index tài liệu

1
Vào trang RAG
2
Upload file

Hỗ trợ .pdf, .txt, .md. Kéo thả hoặc click chọn.

3
Chờ indexing

Xong → file xuất hiện với timestamp.

💡 Index tài liệu kỹ thuật, hướng dẫn, dữ liệu nội bộ. Agent trả lời dựa trên nội dung thực, không phải đoán.

Chương 12LocalLLM — Chạy AI offline qua Ollama Personal Feature

Tích hợp Ollama — chạy AI model trên máy, không cần API key, không phí cloud, hoàn toàn private.

📌 Yêu cầu: Cài Ollama trên máy trước (ollama.com). Workmate AI tự detect.
Thao tácCách làm
Xem models đã càiTrang LocalLLM → danh sách tự load
Tải model mớiNhập tên (vd: llama3.2) → Pull → chờ progress
Xóa modelClick Delete bên cạnh model

Dùng trong Chat: Settings → Providers → thêm Ollama provider → model local xuất hiện trong dropdown chọn model.

💡 Gợi ý model theo RAM:
8GB → llama3.2:3b, phi3:mini
16GB → llama3.2:8b, mistral:7b
32GB+ → llama3.1:70b-q4, qwen2.5:32b

Chương 18Patcher — Anthropic OAuth Bypass Advanced

Hệ thống intercept luồng giao tiếp giữa OpenClaw và Anthropic API, cho phép dùng Claude qua tài khoản Anthropic.com thay vì API key.

⚠️ Tính năng nội bộ nâng cao. Chỉ chỉnh sửa nếu hiểu rõ cơ chế — sai cấu hình có thể làm gián đoạn kết nối.
Workmate AI → OpenClaw → LLM Patcher Proxy → Anthropic API │ Intercept & rewrite: • HTTP headers (xóa fingerprint) • Tool names (bypass signature check) • System prompt content • Response stream
CategoryChức năng
systemRewrite nội dung system prompt
tool_nameĐổi tên tool để bypass signature check
tool_descRewrite mô tả tool
responseXử lý response từ API
headerXóa/thêm HTTP headers

Settings → Patcher → text editor: enable/disable từng rule, chỉnh sửa pattern (hỗ trợ Regex).

Chương 20Quy trình Release

2

git push origin main --tags
3
Chuyển repo sang Public trên GitHub

Chương 22Xử lý sự cố

Gateway không khởi động

  • Kiểm tra port 18789: lsof -i :18789 (Mac/Linux) hoặc netstat -ano | findstr 18789 (Windows)
  • Xem logs: Settings → Developer → View Logs
  • Dashboard → Stop → Start để restart thủ công

Agent không trả lời

  • Kiểm tra API key: Settings → Providers → Validate
  • Kiểm tra Gateway đang Running (Dashboard)
  • Thử model khác để loại trừ model-specific issue
  • Clear session và tạo mới

Kênh mất kết nối

  • WhatsApp / Zalo: QR hết hạn → Request QR mới
  • Telegram: Kiểm tra Bot Token còn active
  • Restart Gateway sau khi cập nhật credentials

Auto-update không hoạt động

  • Kiểm tra repo đang Public trên GitHub
  • Settings → Updates → Check Now (kiểm tra thủ công)
  • Xác nhận đúng Update Channel (stable/beta/dev)
